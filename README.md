História do JAVA

Surgimento da linguagem 
de programação:

A maneira como computamos as coisas sempre evoluiu junto com o que a gente precisa. Lá atrás, o tear automático de Joseph Marie Jacquard já mostrava um passo grande na automatização das fábricas, usando cartões furados para criar os desenhos nos tecidos. Mais tarde, Charles Babbage criou a Máquina Analítica, que podia fazer contas e guardar informações, sendo vista como a "mãe" dos computadores de hoje.

Ada Augusta Lovelace, a filha de Lord Byron, sacou o potencial da Máquina Analítica e criou umas instruções para ela funcionar, virando a primeira pessoa a pensar em lógica de programação. Só que, por muito tempo, ninguém deu bola para o que ela fez, e só em 1953 que o governo americano reconheceu sua importância, dando o nome dela para uma linguagem de programação, chamada Ada.

Na Segunda Guerra, a computação deu um salto gigante com a criação do ENIAC, o primeiro computador eletrônico usado para calcular onde as bombas caíram. No começo, programar era mexer nos componentes do computador, o que era bem complicado, mas depois surgiram linguagens de programação mais fáceis de usar. Aí apareceram linguagens como Cobol, para o comércio, e Pascal e Fortran, para a ciência. Como precisavam de mais gente que soubesse programar, as faculdades começaram a colocar programação nos cursos de computação.

Vendo como é difícil ensinar programação, as universidades estão procurando maneiras melhores de fazer isso. Ninguém sabe qual é a melhor forma, mas tem muita gente estudando o assunto para tentar diminuir o número de alunos que desistem. As pesquisas mostram que o problema maior é que os alunos não têm uma boa base em matemática e não conseguem resolver problemas, o que atrapalha na hora de aprender a programar.

Para melhorar isso, as pessoas têm sugerido várias ideias, como ensinar usando problemas e usar ferramentas interativas e jogos educativos. Algumas escolas estão criando ambientes virtuais que mostram problemas do mundo real, para que o aprendizado seja mais interessante. As pesquisas dizem que juntar essas ideias novas com ferramentas digitais pode ajudar os alunos a terem menos dificuldades.

Para finalizar, a programação é fundamental para a informática e precisa seguir se transformando. Contudo, o aprendizado dela ainda é complicado, com muitos alunos abandonando o curso por não entenderem as ideias complexas. Mesmo sem uma resposta definitiva, as universidades buscam formas de deixar o ensino de programação mais fácil e produtivo. A meta principal é capacitar programadores competentes para as necessidades do universo digital.



Surgimento da linguagem 
Java:

A jornada do Java teve seu ponto de partida em 1991, quando James Gosling e seu time na Sun Microsystems deram início ao projeto Green. A ideia central era prever os avanços da tecnologia e criar uma linguagem de programação sob medida para dispositivos eletrônicos voltados ao consumidor. O propósito primordial era edificar uma linguagem capaz de unificar a comunicação entre variados dispositivos, operando de forma independente da plataforma utilizada. Essa antevisão de um universo conectado por aparelhos, algo hoje corriqueiro com as casas inteligentes, motivou o projeto desde o início.
A linguagem, primeiramente batizada de GreenTalk, derivou do C++ e foi pensada para comandar dispositivos como televisores e controles remotos. Nesse contexto, surgiu o StarSeven, um controle remoto com tela sensível ao toque, concebido para intensificar a interação entre aparelhos domésticos. Durante o processo, o mascote Duke ganhou vida e, posteriormente, se consagrou como o símbolo do Java. Contudo, o projeto StarSeven não alcançou o sucesso almejado, em parte por estar muito à frente de seu tempo.
Diante da ascensão da internet no começo dos anos 90, impulsionada pela popularização do protocolo HTTP e do navegador Mosaic, a equipe Green percebeu o potencial de adaptar sua linguagem para a criação de aplicações web. Foi nesse instante que o projeto foi rebatizado para Oak (Carvalho), em alusão à árvore visível da janela de Gosling. Entretanto, como o nome já estava registrado, o projeto passou por mais uma mudança, dando origem ao Java, inspirado no café Java, uma bebida forte consumida pela equipe durante as sessões de brainstorming.
Em 1995, a linguagem Java foi oficialmente lançada com o conceito inovador de WORA (Write Once, Run Anywhere), traduzido como "escreva uma vez, execute em qualquer lugar". Isso significava que o código poderia ser escrito uma única vez e rodar em qualquer plataforma, graças à máquina virtual Java (JVM). Tal conceito revolucionou o cenário, já que, antes, para executar um programa em diferentes sistemas operacionais, era preciso criar códigos específicos para cada plataforma.
A popularidade do Java foi assegurada por sua portabilidade e pela facilidade com que os programadores conseguiam criar aplicativos para a web. Em 1996, o Java Development Kit (JDK) foi lançado, oferecendo um conjunto completo de ferramentas para os desenvolvedores programarem em Java. Esse lançamento foi fundamental, impulsionando a rápida disseminação do Java, especialmente devido aos applets, pequenos programas executados diretamente no navegador.
A partir de 1997, com o lançamento do Java 2, a linguagem foi dividida em edições distintas, como Java SE (Standard Edition), Java EE (Enterprise Edition) e Java ME (Micro Edition), suprindo diferentes necessidades e tipos de dispositivos. O Java ganhou ainda mais relevância e se firmou como uma das linguagens de maior importância para o desenvolvimento de aplicações empresariais e sistemas móveis.
Desde que a Oracle Corporation comprou a Sun Microsystems em 2008, ela tem se dedicado a dar suporte e evoluir o Java. Essa linguagem permanece entre as mais populares globalmente, contando com uma grande comunidade de programadores e um ecossistema vasto de recursos e estruturas. Atualmente, o Java é essencial para criar sistemas corporativos, aplicativos para celular e soluções complexas, firmando seu lugar como uma das linguagens de programação mais poderosas e importantes do setor.


        Características do Java:
Simples: O aprendizado da linguagem de programação Java pode ser feito em um curto espaço de tempo;
Orientada a objetos: Desde o início do seu desenvolvimento esta linguagem foi projetada para ser orientada a objetos;
Familiar: A linguagem Java é muito familiar para os programadores C/C++ ; 
Robusta: Ela foi pensada para o desenvolvimento de softwares confiáveis, provendo verificações tanto em tempo de execução quanto compilação, o coletor de lixo responsabiliza-se pela limpeza da memória quando houver necessidade;
Segura: Aplicações Java são executadas em ambiente próprio (JRE) o que inviabiliza a intrusão de código malicioso;
Portável: Programas desenvolvidos nesta linguagem podem ser executados em praticamente qualquer máquina desde que esta possua o JRE instalado;



				      Versões Java
JDK 1.0 (1996): Primeira versão;
JDK 1.1 (1997): Adição das bibliotecas JDBC, RMI etc;
J2SE 1.2 (1998): Playground - A partir daqui todas as versões Java foram denominadas de Java 2 Standard Edition, passaram a ter apelidos (Playground) e foi adicionado o Framework Collections etc;
J2SE 1.3 (2000): Kestrel - Inclusão das bibliotecas JNDI, JavaSound etc;
J2SE 1.4 (2002): Merlin - Palavra reservada “assert”, biblioteca NIO etc;
J2SE 5.0 (2004): Tiger - Apesar da versão ser 1.5, agora é chamada apenas de 5. Adições importantes como: Enumeração, Autoboxing, Generics, for-each etc; 
JSE 6 (2006): Mustang -  Entre outras alterações houveram mudança na nomenclatura (remoção do 2 – J2SE) e melhora significativa na performance;
JSE 7 (2011): Suporte ao uso de strings em condições do switch, inferência na criação de objetos com tipos genéricos, simplificação na invocação de métodos com parâmetros varargs e tipos genéricos, gerenciamento automático de recursos (banco de dados, por exemplo) e possibilidade de tratar diversas exceções em um mesmo catch;
JAVA 8 (2014): Trouxe um marco significativo na evolução da linguagem Java, introduzindo expressões lambda e a Streams API, que revolucionaram a forma como lidamos com coleções de dados e processamento paralelo;
JAVA 9 (2017): A modularidade finalmente chegou com o Projeto Jigsaw, permitindo uma organização mais inteligente e modular de nossos projetos Java;
JAVA 10 (2018): Inferência de tipo de variável local, simplificou nossa escrita de código e o tornou mais limpo e legível;
JAVA 11 (2019): Novo HTTP Client, o que proporcionou uma maneira mais moderna e eficiente de lidar com comunicações HTTP em nossos aplicativos Java;
JAVA 12-13 (2019): O Java 12 trouxe aprimoramentos nas expressões switch e formatação de números no padrão Short que levou a melhorias significativas na legibilidade do código.Com o lançamento do Java 13, foram introduzidos blocos de texto para facilitar a manipulação de strings multilinhas, juntamente com melhorias na API de fluxo de processo, proporcionando mais praticidade na manipulação de processos nativos;
JAVA 14-15 (2020): O Java 14 introduziu Records para criar classes de dados imutáveis de forma concisa, além de implementar o pattern matching para instanceof e melhorias na depuração de erros.Com o lançamento do Java 15, foram introduzidas classes ocultas e interfaces seladas, o que garantiu mais controle sobre quem pode herdar nossas classes e interfaces;
JAVA 16-17 (2021): Mais robustez com os Records e o pattern matching, juntamente com a API de acesso a memória externa, que abre portas para integração com código nativo de forma mais segura e eficiente;
JAVA 18 (2022): Trouxe a API de Vetor e UTF-8 por padrão, e assim ganhamos mais eficiência ao lidar com vetores e menos dores de cabeça com codificação;
JAVA 19-21 (2023): Foram introduzidas funções e memória externa, juntamente com Threads virtuais, o que tornou o Java cada vez mais amigável para integração com código nativo e para lidar com concorrência.O Java 20 trouxe um switch case melhorado e projetos Loom e Panama; isso significa menos bugs e mais eficiência na escrita.Com o Java 21, foram introduzidos Record Patterns e recursos de visualização. Assim transformamos a maneira como manipulamos dados;
JAVA 22 (2024): O Java 22 trouxe aprimoramentos no coletor de lixo G1 e API de Funções e Memória Estrangeira.A ideia de lançar programas de código-fonte multi-arquivo sem complicações foi implementada, simplificando ainda mais o desenvolvimento em Java;

Bibliotecas:
Framework Collections: através de interfaces, são definidos e organizados os mais diversos tipos de coleções, de forma que são oferecidas classes que implementam os mais variados padrões de coleções,
JDBC: É uma API que reúne conjuntos de classes e interfaces escritas na linguagem Java na qual possibilita se conectar através de um driver específico do banco de dados desejado.
RMI: é um mecanismo para permitir a invocação de métodos que residem em diferentes máquinas virtuais Java
JNDI: NDI é uma API utilizada em aplicações que acessam recursos externos, ela permite obter esses recursos através do nome.
JavaSound: uma ferramenta que permite manipular áudio em aplicações Java. Ela é uma API de baixo nível que controla a entrada e saída de áudio e MIDI.
NIO: Foi criado para complementar a API Java IO, oferecendo uma alternativa escalável para aplicações que fazem uso intenso de I/O de forma concorrente;
Autoboxing: é um recurso do compilador Java que converte automaticamente tipos primitivos de dados em objetos. Por exemplo, converter um int em um Integer.


Legenda:
JRE: O Java Runtime Environment contém tudo aquilo que um usuário comum precisa para executar uma aplicação Java (JVM e bibliotecas), como o próprio nome diz é o “Ambiente de execução Java”;
JDK: O Java Development Kit é composto pelo JRE e um conjunto de ferramentas úteis ao desenvolvedor Java. Inclui compilador, depurador, desmontador, entre outros;
SDK: Um kit de desenvolvimento de software (SDK) é um conjunto de ferramentas de criação específicas da plataforma para desenvolvedores;
JVM: Compilar o código em Java, Execução de pilhas, Gerenciamento de memória, Execução de threads, entre elas o Garbage Collector, que faz uma limpeza na memória virtual quando existe muito lixo.
Applets: é um pequeno programa de computador que é incorporado em uma página da web ou em outro aplicativo. Ele pode executar tarefas específicas, como gráficos animados, jogos, simulações, entre outros;
Expressões Lambda: Função anônima que pode ser passada como argumento ou armazenada em uma variável;
Streams API: Fornece uma maneira poderosa e concisa de lidar com coleções de dados em Java. Pessoas desenvolvedoras podem realizar operações de processamento de dados, como filtragem, mapeamento e redução de forma declarativa e paralela — o que torna um código mais limpo e eficiente;
Varargs: Ela permite que um método aceite um número indefinido de parâmetros de um tipo específico, tratando-os como se fossem um array comum;
Generics: Introduzidos no Java 5, permitem que classes, interfaces e métodos operem com tipos de dados genéricos, aumentando a segurança de tipo e reduzindo a necessidade de casting;
Enumeração (Enums): Um tipo especial de classe que representa um grupo de constantes. Introduzido no Java 5, facilita a definição de conjuntos fixos de valores;
Autoboxing/Unboxing: Conversão automática entre tipos primitivos (como int) e suas classes wrapper correspondentes (como Integer). Introduzido no Java 5;
Garbage Collector: Um mecanismo da JVM que gerencia a memória automaticamente, liberando objetos não utilizados para evitar vazamentos de memória;
Modularidade (Projeto Jigsaw): Introduzido no Java 9, permite organizar o código em módulos independentes, melhorando a escalabilidade e a manutenção de aplicações grandes;
Records: Introduzidos no Java 14, são classes imutáveis que simplificam a criação de objetos para armazenar dados, reduzindo a verbosidade do código;
Pattern Matching: Recurso que simplifica a verificação de tipos e extração de dados. Introduzido para instanceof no Java 14 e expandido para switch em versões posteriores;
Threads Virtuais: Introduzidos no Java 19, são threads leves gerenciadas pela JVM, permitindo concorrência em larga escala com menor custo de recursos;
API de Memória Externa: Facilita a integração com código nativo e o acesso a memória fora do heap da JVM. Introduzida no Java 16 e aprimorada em versões posteriores;
Switch Expressions: Melhorias nas expressões switch, introduzidas no Java 12, permitindo que o switch retorne valores e simplificando a sintaxe;
Text Blocks: Introduzidos no Java 13, permitem a criação de strings multilinhas de forma mais legível e prática;
Sealed Classes (Classes Seladas): Introduzidas no Java 15, permitem controlar quais classes podem herdar ou implementar uma classe ou interface, aumentando a segurança do código.;
Vector API: Introduzida no Java 18, permite operações eficientes em vetores, otimizando cálculos numéricos e científicos;
UTF-8 por Padrão: A partir do Java 18, o UTF-8 tornou-se o padrão para codificação de caracteres, simplificando a manipulação de textos.;
Record Patterns: Introduzidos no Java 21, permitem a desestruturação de objetos Record em padrões, facilitando a manipulação de dados;
Project Loom: Focado em melhorar a concorrência no Java, introduzindo threads virtuais e outras melhorias para simplificar o desenvolvimento de aplicações concorrentes;
Project Panama: Facilita a integração entre Java e código nativo, melhorando a interoperabilidade com bibliotecas escritas em C/C++;
